# Telegram Bot

## Description

This module provides the notification layer of the monitoring system.

The bot reads the JSON report generated during the analysis phase and allows users to query the system status through Telegram commands.

The JSON report is generated by the PowerShell analysis script and contains the normalized telemetry collected from the system logs.

---

## Requirements

Before running the bot, ensure that:
* A valid `.env` file exists.

Example `.env`:

```text
TELEGRAM_BOT_TOKEN=YOUR_BOT_TOKEN
```

Please run
```text
sudo chmod +x setup.sh
sudo chmod +x main.sh
sudo ./setup.sh
```
To correctly setup the project and then run the bot :)


---

## Running the bot

The bot is started automatically by the project pipeline:

```bash
./main.sh --mock
```

or

```bash
./main.sh --no-mock
```

---

# Available Commands

## `/start`

Displays the list of available commands.

Example:

```text
/start
```

---

## `/summary`

Displays a summary of the analyzed logs.

The summary includes:

* Total number of logs
* Critical events
* Warning events
* Informational events

Example:

```text
/resumen
```

---

## `/critical [n]`

Displays the most recent critical logs.

If no number is provided:

* Default: **5 logs**

Maximum:

* **20 logs**

Examples:

```text
/critical
```

```text
/critical 10
```

---

## `/warning [n]`

Displays the latest warning logs.

Default:

* **5 logs**

Maximum:

* **20 logs**

Examples:

```text
/warning
```

```text
/warning 8
```

---

## `/info [n]`

Displays the latest informational logs.

Default:

* **5 logs**

Maximum:

* **20 logs**

Examples:

```text
/info
```

```text
/info 15
```

---

## `/all`

Displays:

1. A global summary of all analyzed logs.
2. The last **10** processed log entries.

Example:

```text
/all
```


# Project Architecture

* **Bash**

  * Generates mock telemetry.
  * Collects system log data.
  * Produces the raw CSV file.

* **PowerShell**

  * Reads the CSV.
  * Cleans and normalizes the data.
  * Evaluates warning and critical thresholds.
  * Produces the JSON report.

* **Python**

  * Reads the JSON report.
  * Filters the requested information.
  * Formats user-friendly messages.
  * Sends responses through Telegram.

---

# Notes

* The bot always reads the latest generated `analysis.json`.
* Log filtering is performed in Python, not in PowerShell.
* Commands that accept a numeric parameter limit the result to a maximum of **20** logs.
* If no parameter is provided, the default number of logs displayed is **5**.
